# Sample customer RFP

## Section 6.2 Access control

The vendor must describe:
- Supported single sign-on protocols.
- Whether SCIM 2.0 provisioning is available.
- Whether administrators require multi-factor authentication.
- How least-privilege access is enforced.

## Section 8 Incident response

The vendor must describe:
- Customer notification timeline for security incidents involving customer data.
- Recovery time objective (RTO).
- Recovery point objective (RPO).
- Whether incident-response tabletop exercises are performed annually.

## Section 10 Data handling

The vendor must describe:
- Whether a data processing agreement is available.
- How subprocessors are disclosed.
- What regional hosting or data residency options exist.
- What cross-border transfer mechanism is used.

## Section 12 Auditability

The vendor must describe:
- Which actions appear in the audit log.
- Audit log retention period.
- Whether configuration changes and export events are captured.
